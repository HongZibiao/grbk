# 修复博客页面加载超时问题

## 问题分析
- 部署成功（状态为 Ready）
- 页面访问时一直加载不出来
- 可能原因：数据库连接超时或环境变量配置问题

## 解决方案

### 1. 添加数据库连接超时处理
修改 `src/app/page.tsx` 中的 `getPosts` 函数，添加超时处理，确保即使数据库连接失败，页面也能正常显示。

### 2. 优化数据库连接配置
修改 `src/db/index.ts`，确保数据库连接配置正确，避免连接超时。

### 3. 检查环境变量配置
确保 Vercel 中的 `POSTGRES_URL` 环境变量配置正确，使用有效的数据库连接字符串。

### 4. 增加错误日志
在 API 路由中添加错误日志，便于排查问题。

## 具体修改

1. **修改 `src/app/page.tsx`**：
   - 添加 Promise.race 实现超时处理
   - 确保即使数据库连接失败也能返回空数组

2. **修改 `src/db/index.ts`**：
   - 确保数据库连接配置正确
   - 添加错误处理

3. **修改 `src/app/api/init-db/route.ts`**：
   - 增加详细的错误日志
   - 确保数据库初始化过程不会卡住

## 预期结果
- 页面能够正常加载，即使数据库连接失败
- 显示 "暂无文章" 的提示信息
- 可以正常访问管理后台
- 数据库初始化 API 能够正常响应
